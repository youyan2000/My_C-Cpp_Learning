# 数组
数组是固定大小的相同类型元素的顺序集合，本质是一块连续的内存空间
数组为我们提供了一种管理一大堆数据集合的方案
所有的**索引 index**从0开始

## 一维数组（排列）
### 声明
基本格式：`数据类型 数组名[数组大小]`;
e.g.
- `int arr[5];` 
- `char str[10];`
- `float score[30];`

### 访问
#### 指针访问
格式：`数组名[索引值]`
e.g.
```c
int arr[5] = {10, 20, 30, 40, 50};
printf("%d", arr[0]); // 输出第一个元素：10
printf("%d", arr[3]); // 输出第四个元素：40
arr[2] = 300; // 修改第三个元素的值为300
```

#### 索引访问
核心原理：数组名本质是数组首元素的地址（即&arr[0]），指针访问就是通过地址偏移获取元素。
格式：`*(数组名 + 偏移量)`（偏移量 = 索引值）
e.g.（等价于指针访问）
```c
int arr[5] = {10, 20, 30, 40, 50};
printf("%d", *arr); // 等价于arr[0]，输出10（arr是首地址，*arr取首元素值）
printf("%d", *(arr + 3)); // 等价于arr[3]，输出40（地址偏移3个int长度）
*(arr + 2) = 300; // 等价于arr[2] = 300，修改元素
```

### 初始化/定义
- 方式1：完全初始化（赋值个数 = 数组大小）       
e.g.`int arr[5] = {1, 2, 3, 4, 5};` // 数组元素：arr[0]=1, arr[1]=2, ..., arr[4]=5

- 方式2：部分初始化（赋值个数 < 数组大小）   
e.g.`int arr[5] = {1, 2};` // 前2个元素赋值，剩余元素默认是0（arr[2]=0, arr[3]=0, arr[4]=0）    

- 方式3：省略数组大小（由初始化元素个数自动推断）
e.g.`int arr[] = {1, 2, 3, 4, 5};` // 自动推断数组大小为5，等价于int arr[5]

## 二维数组（方阵）
### 声明
基本格式：`数据类型 数组名[行数][列数];`
e.g.
- `int arr[2][3];` // 声明一个2行3列的二维数组，共2×3=6个元素
- `char map[5][5];` // 声明一个5行5列的char类型二维数组（模拟地图）
- `float mateix[3][4];` // 声明一个3行4列的浮点型二维数组（矩阵）

### 访问
#### 索引访问
格式：`数组名[行索引][列索引]`（行、列索引均从0开始，最大行索引=行数-1，最大列索引=列数-1）
e.g.
```c
int arr[2][3] = {{1,2,3}, {4,5,6}};
printf("%d", arr[0][0]); // 输出第一行第一列：1
printf("%d", arr[1][2]); // 输出第二行第三列：6
arr[0][2] = 30; // 修改第一行第三列元素为30
```

#### 指针访问
核心原理：二维数组名是“首行地址”（即&arr[0]，arr[0]是第一行的一维数组名），通过两次地址偏移访问元素。
格式：`*(*(数组名 + 行偏移) + 列偏移)`
e.g.（和上面索引访问等价）
```c
int arr[2][3] = {{1,2,3}, {4,5,6}};
printf("%d", *(*(arr + 0) + 0)); // 等价于arr[0][0]，输出1
printf("%d", *(*(arr + 1) + 2)); // 等价于arr[1][2]，输出6
*(*(arr + 0) + 2) = 30; // 等价于arr[0][2] = 30
```

### 初始化/定义
- 方式1：按行初始化（推荐，清晰直观）     
e.g.`int arr[2][3] = {{1, 2, 3}, {4, 5, 6}};` //第一行元素{1,2,3}，第二行元素{4,5,6}，对应arr[0][0]=1，arr[1][2]=6
      
- 方式2：连续初始化（不分行，按顺序赋值）    
e.g.`int arr[2][3] = {1, 2, 3, 4, 5, 6};` //编译器自动按“行优先”分配，和方式1完全等价

- 方式3：部分初始化（未赋值元素默认0），可省略行数   
e.g.`int arr[][3] = {{1, 2}, {4}};` //省略行数（自动推断为2行），第一行{1,2,0}，第二行{4,0,0}

## 动态数组 Dynamic Array
数组本质上就是一块连续的内存空间
因此，我们可以利用指针，通过申请一块地址来注册一个数组

### 初始化/定义
C中形如 : `int *arr = (int *)malloc(n * sizeof(int));`
C++中形如 : `int* arr = new int[5];`

### 扩容
数组大小翻倍 :  `arr = (int*)realloc(arr, 2 * n * sizeof(int));`

### 析构
释放内存 + 置空指针
```c
free(arr);
arr = NULL;
```

### 普通数组与动态数组对比
| 对比项     | 普通数组                    | 动态数组                                        |
| ---------- | --------------------------- | ----------------------------------------------- |
| 内存位置   | **栈内存 Stack**            | **堆内存 Heap**                                 |
| 分配方式   | 系统自动分配、自动释放      | 手动 malloc/calloc/realloc 分配，手动 free 释放 |
| 分配时机   | 编译/函数执行时自动确定     | 程序运行时动态申请                              |
| 内存大小   | 较小，有固定上限            | 较大，受限于系统总内存                          |
| 生命周期   | 函数/代码块结束**自动回收** | 直到调用 free 才释放                            |
| 访问方式   | 快                          | **内存间接寻址 memory indirection**             |
| 访问成本   | 无                          | 频繁申请释放易产生内存碎片                      |
| 空间连续性 | 连续内存                    | 不一定连续                                      |
| 越界后果   | 栈溢出，程序崩溃            | 踩内存、逻辑异常、崩溃                          |

## C++11标准数组`std::array`
特性：
1. 边界检查
2. 记录数组大小

## 其他神奇特性
1. **内存访问违规 memory accsee violation** : 当你的索引指向了不属于你的内存
2. 数组的**遍历 traversal**通常由for循环实现
3. 未赋值的元素默认是0（全局数组）或随机值（局部数组）
4. 二维数组的列数不可省略
5. 二维数组和一维数组在内存上并没有什么区别，都是一块连续的内存空间
6. 计算数组大小的常用手法（当然以int数组arr为例） : `sizeof(arr) / sizeof(int)`
7. 数组大小必须是常量（不能是变量，宏定义当然没关系），如果是变量，必须这样定义 : `static const int size = `
