# 实验室成员管理系统

## 目的:
开发一个实验室成员管理系统，取代结构体类型，定义教师类和学生类。定义实验室类，将所有教师和学生作为成员。

## 要求：
1. 引入类、对象、构造函数等面向对象编程因素，通过编程加深对“封装”的认识，掌握类、对象、成员函数、构造函数的定义与使用。
2. 成员数据要获取保护，不能被直接使用。
3. 为教师类和学生类定义一个共同的基类 Member，将它们共同的成员数据和函数封装到基类 Member 中。教师类和学生类作为 Member 的派生类，它们的成员数据和成员函数可能需要做相应修改，应尽量复用 Member 类的代码。
4. 仅限使用 C++ 类进行面向对象编程，代码内容使用 C 完成，禁止使用 CIN、COUT 这些 C++ 的函数。严禁抄袭！

## 具体功能：
- 菜单显示、菜单选择、添加成员、根据姓名查询成员信息、根据姓名删除成员
- 显示所有教师或学生（按职位、组别排序）、统计教师和学生数量、退出系统等。
- 注意：所有功能要有判别错误的功能，并能对输入错误信息进行提示；各功能使用过程中要有适当提示。
- 学生信息包括：姓名、性别、专业、组别、职位、学号；
- 教师信息包括：姓名、性别、专业、职位（指导教师）等。
- （所有成员信息为 IFR 实验室实际信息）

## 提示：
所有数据存储于文件中，启动读取文件中数据，退出时保存数据到文件中  
文件格式：csv，具体格式查看所给模板，分别用 excel，记事本打开即可了解 csv 文件保存格式，也可上百度进行搜索该格式  
文件数据保存显示时要求所有教师或学生按组别、职位排序。
