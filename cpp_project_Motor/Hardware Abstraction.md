# 虚函数与多态：多态与硬件抽象 —— “万能电机接口”

## 【背景故事】

在备赛中，硬件方案经常变。今天云台是GM6020电机，明天就可能是DM4310，或者我们要在一个没有硬件的纯软件环境中调试算法。

如果你的控制代码里写满了 `can_send(...)`，每次换电机都要重写底盘代码，这简直是灾难。

解决办法是：**定义一个“标准”，所有电机都必须遵守这个标准**。上层代码只对着“标准”发号施令。

---

## 【任务目标】

利用 C++ 的 **虚函数 (Virtual Function)** 和 **继承 (Inheritance)**，设计一个通用的电机驱动框架。你需要实现一个统一的接口，并派生出“模拟硬件电机”和“纯软件虚拟电机”两个版本，最后用同一个测试函数驱动它们。

---

## 【涉及知识点】

- **抽象基类**: virtual 关键字与纯虚函数 = 0。
- **多态性**: 基类指针指向派生类对象（Base* ptr = new Derived()）。
- **虚析构函数**: virtual ~Base() {} 的重要性。
- **覆盖**: override 关键字。

---

## 【详细步骤】

### 步骤 1: 定义“万能标准”（Abstract Base Class）

创建一个名为 `IMotor` 的抽象基类。它规定了任何一只电机都必须具备的能力：

1. **设置输出**: 告诉电机用多少力（或电流）转动。
2. **读取反馈**: 询问电机当前转速是多少。
3. **刷新状态**: 模拟电机内部的控制周期。

注意：基类只管“画大饼”，不负责实现（使用纯虚函数）。

### 步骤 2：实现“真实硬件电机”（Derived Class A）

创建一个 `DJIMotor` 类，继承自 `Motor`。

- **模拟场景**：假设这是一个真实的大疆电机，它通过 CAN 总线通信。
- **实现逻辑**：
  - 在 `SetOutput` 中，用 `printf` 打印模拟发送 CAN 报文，例如 `[CAN BUS] ID: 0x201, Send Current: 500`。
  - 在 `GetSpeed` 中，返回一个模拟值（比如固定返回 1000 rpm）。

### 步骤 3：实现“物理仿真电机”（Derived Class B）

创建一个 `SimMotor` 类，继承自 `Motor`。

- **模拟场景**：这是一个运行在电脑里的虚拟电机，它有物理惯性。
- **实现逻辑**：
  - 定义一个成员变量 `velocity_` 存储当前速度。
  - 在 `Update` 函数中实现简单的物理模型：新速度 = 旧速度 + (输入力 - 阻力)。这样当给到输入时，它的速度会慢慢上去，而不是瞬间到达。

### 步骤 4：编写“无差别操作”的测试函数

编写一个全局函数 `void TestMotor(Motor* motor)`。

- 这个函数完全不知道传入的是 DJI 电机还是仿真电机。
- 它只管调用 `SetOutput` 让电机动起来，然后调用 `Update` 和 `GetSpeed` 查看效果。
