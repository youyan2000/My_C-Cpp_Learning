# 可变数组 Resizable Array / Dynamic Array
C++中我们通过`std::vector`实现动态数组
她是 C++ 标准库中最常用的序列容器之一，封装了动态数组，提供了自动内存管理、随机访问、尾部高效增删等特性

## 基本语法
### 定义
```cpp
std::vector<int> v;
```
### 添加元素
```cpp
v.push_back(1);                // 尾部添加
v.emplace_back(2, 3);          // 原地构造，避免拷贝（如元素是复杂类型）
v.insert(v.begin(), 0);        // 在开头插入 0（效率低）
```

### 访问元素
```cpp
int a = v[0];                  // 不检查边界
int b = v.at(0);               // 检查边界，越界抛 std::out_of_range
int front = v.front();         // 首元素
int back = v.back();           // 尾元素
```

### 删除元素
```cpp
v.pop_back();                  // 删除尾部元素
v.erase(v.begin());            // 删除指定位置元素
v.clear();                     // 清空所有元素（size=0，capacity 不变）
```

### 容量操作
```cpp
size_t sz = v.size();          //实际存储元素个数
size_t cap = v.capacity();     //已分配内存能够容纳的元素个数上限
v.reserve(100);                // 请求预留至少能容纳100个元素的内存空间
v.shrink_to_fit();             // 请求将容量缩减到尽可能接近当前大小，释放多余内存（C++11）
```

### 迭代器
```cpp
for (auto it = v.begin(); it != v.end(); ++it) { /* ... */ }
/* 或者 */ 
for (int x : v) { /* ... */ } 
```

## 优化使用
当我们向vector里增加元素时，我们需要新给他分配一个空间，并把旧的全部内容复制到新的位置，然后删除旧位置的内存
换而言之，使用`push_back`和进行扩容是非常浪费性能的
所以我们应该先用`reserve`分配好所需要的内存空间，再用`emplace_back`直接传入数据构造（而非先构造在复制进去）

## 与其他容器的对比
| 容器          | 随机访问 | 中间插入/删除 | 内存连续性 | 适用场景                          |
| ------------- | -------- | ------------- | ---------- | --------------------------------- |
| `std::vector` | O(1)     | O(n)          | 是         | 多数默认选择，尾部操作为主        |
| `std::deque`  | O(1)     | O(n)但更快    | 分块连续   | 头尾操作频繁                      |
| `std::list`   | O(n)     | O(1)          | 否         | 频繁中间插入/删除，不需要随机访问 |
| `std::array`  | O(1)     | 不支持        | 是         | 固定大小，编译期确定              |
